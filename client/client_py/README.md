# server-client交互流程说明

在 HTTP 协议中，GET 和 POST 是两种最常用的请求方法：

- **GET 方法**：
	- 用于请求服务器发送特定资源
	- 请求参数附加在 URL 后面（查询字符串）
	- 有长度限制（因浏览器而异）
	- 可以被缓存、保留在浏览器历史记录中
	- 只应用于数据查询，不应用于数据修改
- **POST 方法**：
	- 用于向服务器提交数据
	- 请求参数包含在请求体中
	- 没有长度限制
	- 不会被缓存、不会保留在浏览器历史记录中
	- 适用于创建/更新资源或提交敏感数据

目前的server-client模式为server负责游戏逻辑，返回游戏当前局面；client负责根据server返回的游戏局面进行决策，并返回该步所做的决策。

1. **启动服务器**：
	- 在终端中运行 fake_server.py，服务器开始监听 http://localhost:5001。
2. **启动客户端**：
	- 在另一个终端中运行 `Program.py`，客户端启动后主动向服务器发送 `GET` 请求获取初始局面。
3. **通信流程**：
	- 客户端通过 `GET` 请求获取初始局面。
	- 客户端根据局面生成策略，并通过 `POST` 请求将策略发送回服务器。
	- 服务器接收策略并打印到控制台。
