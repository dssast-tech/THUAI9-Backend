# dev_test 当前架构总览

## 1. 项目目标（当前阶段）

`dev_test` 的作用是：

- 用 Tkinter 做一个后端调试界面。
- 支持两种数据来源：
  - 本地 runtime 环境（直接跑 `Environment`）
  - mock 回放数据（`data/log.json` 和 `data/mock_sets/*.json`）
- 让我们能看见棋盘、棋子、回合信息，方便联调。

## 2. 目录分层（当前）

```text
dev_test/
├─ main.py                 # 命令行入口（非 UI）
├─ core/
│  ├─ data_provider.py     # 数据源选择：runtime / mock
│  ├─ decoder.py           # mock 原始 JSON -> 统一结构
│  └─ events.py            # 轻量事件总线与循环壳
├─ logic/
│  └─ controller.py        # 回合推进与模式管理
├─ ui/
│  ├─ main_ui.py           # 主界面逻辑与交互入口
│  └─ components.py        # 可复用 UI 组件（棋盘/信息区/按钮等）
└─ data/
   ├─ log.json
   └─ mock_sets/*.json
```

## 3. 运行主链路

### 3.1 UI 路径（现在最常用）

1. 启动 `ui/main_ui.py`。
2. `MainUI` 创建 `Controller`，弹窗选择数据源。
3. 点击“加载测试数据”后：
   - runtime：`DataProvider.load_runtime_environment` 返回本地环境配置。
   - mock：读取指定 json，`GameDataDecoder.decode` 统一成 `map/soldiers/rounds`。
4. 点击“开始回放/单步执行”：
   - `Controller.run_round()` 推进 1 回合。
   - UI 刷新棋盘、棋子位置、右侧日志。

### 3.2 命令行路径（调试）

1. 启动 `main.py`。
2. 创建 `Controller` 并尝试加载 runtime。
3. 调用 `controller.run_loop()` 进行回合推进。

## 4. 当前已经打通的能力

- 地图与棋盘渲染：固定展示左上 20x20 视图。
- 格子颜色规则：`1` 为浅绿，其余为深棕。
- 棋子渲染：player1 红色、player2 蓝色，格内白字标签（阵营+序号）。
- 右侧信息展示区：
  - 基础事件日志
  - 回合细节（行动/伤害变化），要增加更多信息
- 回放节奏：自动回放每 1 秒推进 1 回合。

## 5. 当前架构边界

- `manual` 模式可用；`half-auto/auto` 在控制层仍是待开发状态。
- 事件系统是轻量版本（够用，但还不是完整业务总线）。
- runtime 走本地 `Environment`，不是远程后端流式同步模式。
