# core / logic / ui 核心模块说明

## 1. 入口层

### `main.py`

- 用于命令行方式启动调试流程。
- 主要做三件事：
  - 创建 `Controller`
  - 调 `load_game_data`
  - 调 `run_loop`
- 适合做“快速烟雾测试”，不适合看完整可视化。

## 2. core 层

### `core/data_provider.py`

- 负责“数据从哪里来”。
- 关键接口：
  - `get_game_data(prefer_runtime=True/False, mock_dataset=...)`
  - `list_mock_datasets()`
- runtime 模式返回本地环境描述，不直接返回完整回放帧。
- mock 模式读取 json 并附带数据集名。

### `core/decoder.py`

- 负责把 mock 原始数据转成统一结构。
- 当前输出重点字段：
  - `map`（含 `rows`）
  - `soldiers`
  - `rounds`（含 `actions`、`stats`）
- 已保留动作中的 `damageDealt`，用于 UI 显示伤害信息。

### `core/events.py`

- 轻量事件总线：`EventBus`。
- 基础循环壳：`BasicEventLoop`。
- 已定义事件：
  - `GAME_LOADED`
  - `ROUND_STARTED`
  - `ROUND_FINISHED`
  - `GAME_OVER`
  - 以及 loop 事件。

## 3. logic 层

### `logic/controller.py`

- 作用：统一管理环境、数据和回合推进。
- 核心职责：
  - 初始化/重置本地环境
  - 按 source（runtime/mock）执行回合
  - 发布事件到 UI
- 当前状态：
  - `manual` 支持
  - `half-auto`、`auto` 暂未实现

## 4. ui 层

### `ui/components.py`

- 存放可复用界面组件：
  - `InfoPanel`（右侧文本区）
  - `ButtonPanel`
  - `PlayerSummaryCard`
  - `ChessboardPanel`（棋盘绘制核心）

### `ui/main_ui.py`

- 主界面流程编排中心。
- 当前已实现：
  - 数据源选择弹窗
  - 加载/开始/暂停/单步/重置
  - 棋盘与棋子刷新
  - 回合细节日志展示（行动、伤害、血量变化）

## 5. 数据与测试辅助

### `data/`

- `log.json`：默认 mock 样例。
- `mock_sets/*.json`：更多回放样例。

### `tests/debug_tests/`

- `run_runtime_debug.bat`：一键跑 runtime 调试。
- 作用是快速确认环境是否能推进回合。
